Parameterise the .pbit query paths -- one-time migration ​

The v1.0 .pbit files (swao-report.pbit + swao-portfolio.pbit) were authored by per-CSV Get Data > Text/CSV imports. PowerBI baked the absolute path into each query's M code instead of routing through the SWAOExportPath / SWAOPortfolioExportPath parameter. As a result, refreshing against a new assessment bundle requires editing 17 (single-app) or 19 (portfolio) absolute paths -- one per query.

This document is the one-time fix. After applying it, changing the parameter value in PowerBI's Parameter verwalten updates every query in one click.

After the fix, re-save the .pbit. The fix then ships with the template forever.

Recipe per query (identical pattern, 17 or 19 times) ​

In Power Query Editor: open each query -> Start > Erweiterter Editor -> swap the Quelle = line. Leave the Höher gestufte Header, Geänderter Typ, and in lines untouched.

Generic before / after ​

Before (the line that needs changing):

m
Quelle = Csv.Document(File.Contents("C:\Projects\accenture\swao\examples\portfolio-workspace\portfolio\apps\sovereign-health\wsp\exports\2026-05-10T05-34-42\star\<FILENAME>.csv"),[Delimiter=",", Columns=N, Encoding=65001, QuoteStyle=QuoteStyle.None]),

After (the only change is the file argument to File.Contents):

m
Quelle = Csv.Document(File.Contents(SWAOExportPath & "\<FILENAME>.csv"),[Delimiter=",", Columns=N, Encoding=65001, QuoteStyle=QuoteStyle.None]),

The exact Columns=N and QuoteStyle=... values in the existing query stay as PowerBI generated them; do not change those. The only text you replace is the quoted absolute-path string (from "C:\... to .csv") with SWAOExportPath & "\<FILENAME>.csv".

For the portfolio template, the parameter is named SWAOPortfolioExportPath instead of SWAOExportPath.

Worked example -- dim_app ​

Before (your current dim_app query) ​

m
let
    Quelle = Csv.Document(File.Contents("C:\Projects\accenture\swao\examples\portfolio-workspace\portfolio\apps\sovereign-health\wsp\exports\2026-05-10T05-34-42\star\dim_app.csv"),[Delimiter=",", Columns=11, Encoding=65001, QuoteStyle=QuoteStyle.None]),
    #"Höher gestufte Header" = Table.PromoteHeaders(Quelle, [PromoteAllScalars=true]),
    #"Geänderter Typ" = Table.TransformColumnTypes(#"Höher gestufte Header",{{"app_id", type text}, {"name", type text}, {"business_domain", type text}, {"business_criticality", type text}, {"regulatory_class", type text}, {"seven_r_label", type text}, {"modernization_position", type text}, {"coverage_score", Int64.Type}, {"confidence", type text}, {"portability_score", Int64.Type}, {"landing_zone", type text}})
in
    #"Geänderter Typ"

After (paste this in full) ​

m
let
    Quelle = Csv.Document(File.Contents(SWAOExportPath & "\dim_app.csv"),[Delimiter=",", Columns=11, Encoding=65001, QuoteStyle=QuoteStyle.None]),
    #"Höher gestufte Header" = Table.PromoteHeaders(Quelle, [PromoteAllScalars=true]),
    #"Geänderter Typ" = Table.TransformColumnTypes(#"Höher gestufte Header",{{"app_id", type text}, {"name", type text}, {"business_domain", type text}, {"business_criticality", type text}, {"regulatory_class", type text}, {"seven_r_label", type text}, {"modernization_position", type text}, {"coverage_score", Int64.Type}, {"confidence", type text}, {"portability_score", Int64.Type}, {"landing_zone", type text}})
in
    #"Geänderter Typ"

The only change is on the Quelle line: the quoted absolute path becomes SWAOExportPath & "\dim_app.csv". Everything else is identical.

After all 17 (or 19) queries are rewired ​

  1. Start > Schließen und übernehmen. All queries reload from whatever the parameter currently points at.
  2. Verify a card on the report (Latest Assessment is good) shows the new bundle's data (e.g. 11.05.2026 06:02 after pointing at the real-LLM bundle).
  3. Datei > Exportieren > Power BI-Vorlage. Overwrite the existing .pbit at C:\Projects\accenture\swao\docs\templates\powerbi\swao-report.pbit (or swao-portfolio.pbit). The fix is now permanent in the template.
  4. Smoke-test: open the saved .pbit in a fresh PowerBI Desktop session. PowerBI should prompt for the parameter value. Paste a different bundle path and confirm all visuals re-render.

After this, future bundle changes are one paste in Parameter verwalten -- no per-query edits ever again.

Optional polish (do it now or skip) ​

While each query is open in Erweiterter Editor, you can also fix the integer-truncation legacy by changing Int64.Type to type number for columns that hold decimal values. This is purely optional; the dashboards already work because the visual format masks the truncation.

QueryColumns to flip to type number
dim_appcoverage_score, portability_score
dim_evidencereliability_weight
fact_assessmentsscore, threshold
fact_runsduration_minutes, llm_total_cost_usd, lz_weight_sovereign_score, lz_weight_service_coverage, lz_weight_portability, lz_weight_cost_tier

If you flip these, the Coverage % card on Page 1 may need its visual format set to "Percent" so 0.93 displays as 93% rather than 0,93. Other visuals likely handle the change automatically.

If you don't flip them, leave as-is and the current display behaviour stays. Either path ships a working dashboard.

Summary -- minimal effort path ​

  1. Open swao-report.pbit (or your live .pbix).
  2. Daten transformieren.
  3. For each of the 17 queries: Erweiterter Editor, replace the Csv.Document(File.Contents("C:\\...")) argument with File.Contents(SWAOExportPath & "\<filename>.csv"). Fertig.
  4. Schließen und übernehmen.
  5. Datei > Exportieren > Power BI-Vorlage > overwrite swao-report.pbit.
  6. Repeat for swao-portfolio.pbit with the 19 queries and SWAOPortfolioExportPath parameter name.

End state: changing the parameter once updates the whole dashboard. The .pbit ships with this baked in for future operators.
